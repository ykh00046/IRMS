# IRMS 저울 에이전트 (A&D GX · Mettler XP)

계량하는 **현장 PC**에서 도는 작은 트레이 프로그램입니다. 저울(RS-232C)을 읽어
`http://127.0.0.1:8787` 로 무게를 내어주고, 배합 화면의 [저울] 버튼이 그 값으로
실제량을 자동 입력합니다. **파이썬 설치 불필요** — exe 하나로 동작합니다.

- 창 없음: 작업표시줄 오른쪽 아래 **트레이 아이콘**으로 상주
- **부팅 시 자동 실행**: 첫 실행 때 자동으로 켜짐(트레이 메뉴에서 끄기 가능)
- 설정 유지: `%APPDATA%\IRMS-Scale\config.json` (업데이트해도 보존)
- 상태 확인: 트레이 아이콘 우클릭 → "저울: COM3 연결됨" / 로그 폴더 열기

## 설치 (현장 PC, 1회)

1. GX-10202M 뒷면 RS-232C(D-Sub 9핀, **스트레이트 케이블**) ↔ PC.
   시리얼 포트가 없으면 USB-시리얼 컨버터(FTDI 권장) 사용.
2. 관리자가 배포한 `IRMS-Scale` 폴더를 복사(예: `C:\IRMS-Scale\`)
3. `IRMS-Scale.exe` 더블클릭 → 트레이에 아이콘이 뜨면 완료.
   이후 재부팅해도 자동으로 실행됩니다.

## 사용 (배합 화면)

1. 자재를 저울에 올리고 표시가 안정될 때까지 대기
2. 저울의 **PRINT 키**를 누름 → 화면의 다음 미입력 행(또는 커서를 둔 행)에
   실제량이 자동 입력되고 LOT 칸으로 이동
3. 다음 자재도 올리고 PRINT — 행이 순서대로 채워집니다

허용 편차는 자재별 ±0.05 g — 초과하면 저장이 차단됩니다(합계 편차 제한 없음).
PRINT 는 저울 표시가 안정(○ 표시)일 때만 전송됩니다(저울 공장 기본 동작).
저울 없이 쓰는 PC 에서는 기존처럼 실제량을 직접 입력하면 됩니다.

## 트레이 메뉴

| 메뉴 | 설명 |
|---|---|
| 저울: COMx 연결됨 / 연결 안 됨 | 현재 상태 표시 |
| 다시 연결 | 케이블 재연결 후 수동 재시도 |
| 부팅 시 자동 실행 | 체크 토글 (기본 켜짐) |
| 로그·설정 폴더 열기 | `%APPDATA%\IRMS-Scale\` (agent.log, config.json) |
| 종료 | 에이전트 종료 |

## 저울 통신 설정 (config.json)

`%APPDATA%\IRMS-Scale\config.json`:

| 항목 | 설명 |
|---|---|
| `protocol` | `"and"` = A&D(GX-10202M 등, 기본 2400/7/E/1) · `"mt-sics"` = Mettler(XP10002S 등, 기본 9600/8/N/1) |
| `port` | `"COM4"` 처럼 고정 지정. `null` 이면 자동 탐지 |
| `baudrate` 등 | `null` 이면 프로토콜 기본값. 저울 쪽 설정을 바꿨다면 여기서 맞춤 |
| `yield_to` | 같은 저울을 쓰는 **기존 프로그램 exe 이름 목록**(예: `["LabX.exe"]`). 그 프로그램이 실행되면 포트를 자동 양보하고, 종료되면 자동 재연결 |

**저울 여러 대(권장 형식)** — `scales` 목록에 넣으면 동시에 연결되고,
어느 저울에서 PRINT 를 눌러도 화면에 입력됩니다(전환 조작 불필요):
```json
{
  "http_port": 8787,
  "scales": [
    { "name": "GX-10202M", "protocol": "and", "port": "COM4" },
    { "name": "XP10002S", "protocol": "mt-sics", "port": "COM3",
      "yield_to": ["COMBINATION.exe"] }
  ]
}
```
(COMBINATION.exe = '배합 통합프로그램'. 켜지면 XP 포트만 자동 양보,
꺼지면 자동 재연결 — GX 는 영향 없음.)
수정 후 에이전트 재시작(트레이 종료 → exe 재실행).

## 기존 프로그램과 같은 저울을 쓸 때 (yield_to)

시리얼 포트는 한 번에 한 프로그램만 열 수 있습니다. `yield_to` 에 기존
프로그램 exe 이름을 넣으면: 그 프로그램을 켜는 순간(3초 내) 에이전트가
포트를 놓아주고(트레이 상태 "양보 중"), 끄면 자동으로 다시 연결됩니다.
양보 중에는 PRINT 입력이 동작하지 않습니다(기존 프로그램이 사용 중).

## 문제 해결

- **PRINT 를 눌러도 입력 안 됨**: 트레이 아이콘이 떠 있는지("저울: COMx 연결됨"),
  브라우저에서 `http://127.0.0.1:8787/health` 가 열리는지 확인
- **저울: 연결 안 됨**: 케이블·전원 확인, WinCT 등 포트 점유 프로그램 종료 후
  트레이 메뉴 "다시 연결"
- **측정 중 경고만 나옴**: 저울 표시가 안정(ST)되어야 입력됩니다 — 바람막이/진동 확인

## 개발·빌드 (개발 PC)

```
pip install -r requirements.txt
python agent.py --console        # 콘솔 모드로 디버깅
pyinstaller irms_scale.spec --clean --noconfirm   # → dist/IRMS-Scale/
```
