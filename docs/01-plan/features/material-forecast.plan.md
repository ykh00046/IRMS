# 자재 소모량 예측·발주 추천 — Plan

> 누적된 재고 차감 이력으로 원재료별 소모 추세를 분석하고, 예상 소진일과 권장 발주량을 산출해 책임자에게 발주 추천을 제공한다.

## 1. Overview

| 항목 | 값 |
|------|------|
| Feature | `material-forecast` |
| Phase | Plan |
| 작성일 | 2026-06-01 |
| Priority | High (R2 추가기능 1위 추천) |
| Level | Dynamic |

| 선행 완료 | `material-stock-tracking` (2026-04, 재고 추적 + 자동 차감 + 이력 축적) |

## 2. Problem Statement

`material-stock-tracking` 도입으로 **재고 차감 이력(`material_stock_logs`)이 매 계량마다 축적**되고 있다.
그러나 현재 시스템은 "지금 얼마 남았는지"(현재고)와 "임계치 미달 경고"만 제공할 뿐,
**"이대로 쓰면 언제 소진되는지", "지금 얼마를 발주해야 하는지"**는 알려주지 못한다.

당시 `material-stock-tracking` Plan의 **Out of Scope**로 명시된 두 항목이 바로 이번 기능의 범위다:
- 재고 예측 (소비 추세 기반)
- 발주서/발주 추천

### Pain Points

1. **발주 타이밍 의존성** — 임계치 경고는 "이미 부족"해진 뒤에 뜬다. 발주 → 입고까지의 리드타임을 고려하면 늦다.
2. **발주량 산정의 감(感)** — 책임자가 월 소비량을 머릿속으로 추정해 발주량을 정함. 과다발주(재고 비용)·과소발주(작업 중단) 반복.
3. **소진 가시성 부재** — "이 자재 앞으로 며칠 버티나?"를 한눈에 볼 화면이 없음.
4. **데이터 사장(死藏)** — 매일 쌓이는 소모 이력이 경고 트리거 외에는 활용되지 못함.

## 3. Feature Items

### 3.1 소모 추세 분석 (예측 엔진)

| Item | Detail |
|------|--------|
| 데이터원 | `material_stock_logs` 중 `reason='measurement'` (음수 delta = 실제 소모) |
| 분석 기간 | 후행 N일 (기본 30일, 60/90일 선택 가능) |
| 산출값 | 일평균 소모량(avg_daily), 기간 내 총 소모량, 표본 일수 |
| 예측값 | 예상 소진일(predicted stockout date), 잔여 일수(days_remaining) = 현재고 / 일평균 |

### 3.2 발주 추천

| Item | Detail |
|------|--------|
| 발주점(reorder point) | `avg_daily × 리드타임` (+ 안전계수) |
| 권장 발주량 | `avg_daily × 목표 커버리지일 − 현재고` (음수면 0) |
| 추천 상태 | `urgent`(리드타임 내 소진) / `soon`(리드타임+버퍼 내) / `ok` / `no_data`(소모 이력 없음) |
| 파라미터 | 리드타임(기본 7일), 목표 커버리지(기본 30일), 안전버퍼 — 원재료별 설정 가능, 미설정 시 전역 기본값 |

### 3.3 예측·발주 대시보드 (책임자 전용)

| Item | Detail |
|------|--------|
| 위치 | `/management` 신규 탭 "소모예측·발주" |
| 표 컬럼 | 원재료 / 카테고리 / 현재고 / 일평균 소모 / 예상 소진일 / 잔여일수 / 권장 발주량 / 상태 |
| 정렬 | 긴급도순(urgent → soon → ok), 잔여일수 오름차순 |
| 필터 | 분석기간(30/60/90), 상태(발주 필요만 보기) |
| 강조 | `urgent`/`soon` 행 색상 강조 + 상단 요약 배너("발주 권장 N건") |

### 3.4 발주 추천서 내보내기

| Item | Detail |
|------|--------|
| 형식 | CSV (기존 `/stats/export` 패턴 재사용) |
| 내용 | 발주 권장 자재만(또는 전체) + 권장 발주량 + 예상 소진일 |
| 용도 | 구매 담당자 전달용 (외부 발주 시스템에 수기 입력) |

## 4. Scope

### In Scope
- 소모 이력 기반 일평균 소모량·예상 소진일 산출 (forecast 서비스)
- 발주점·권장 발주량·추천 상태 계산
- 원재료별 리드타임/목표 커버리지 설정(컬럼 추가, 전역 기본값 fallback)
- 예측·발주 API (조회 + CSV 내보내기), 책임자 권한
- 관리 화면 신규 탭 + 요약 배너
- pytest 단위/통합 테스트

### Out of Scope
- 외부 ERP/구매 시스템 자동 발주(전송)
- 다중 창고/로트별 예측
- 머신러닝·계절성 모델(단순 이동평균만)
- 트레이 클라이언트 푸시 알림(추후 별도 PDCA)
- 비용/단가 기반 발주 최적화

## 5. Success Criteria

1. 소모 이력이 있는 원재료에 대해 일평균 소모량과 예상 소진일이 정확히 계산된다.
2. 현재고·리드타임·커버리지로 권장 발주량과 추천 상태(urgent/soon/ok)가 산출된다.
3. 소모 이력이 전혀 없는 원재료는 `no_data`로 분류되어 0으로 나누지 않는다.
4. 책임자는 관리 화면에서 발주 임박 자재를 우선순위로 확인하고 CSV로 내보낼 수 있다.
5. 권한: operator는 접근 불가, manager 이상만 조회/내보내기 가능.
6. 기존 재고/계량 기능에 회귀가 없다(전체 pytest 통과).

## 6. Design Decisions (자율 판단)

| 결정 | 선택 | 근거 |
|------|------|------|
| 예측 모델 | 후행 N일 단순이동평균 | 현장 비개발자 운영. 설명가능성·예측가능성 우선. 데이터량도 ML에 부족 |
| 분모 | 분석기간 달력일수(N) | "활성일수"보다 보수적·안정적. 간헐적 사용 자재의 과대추정 방지 |
| 소모 데이터원 | `material_stock_logs` measurement delta | 계량 확정의 단일 진실원. `/stats/consumption`(recipe_items)은 취소분 포함 위험 |
| 파라미터 저장 | `materials`에 컬럼 추가(0=전역기본) | 공급사별 리드타임 상이. 단순 마이그레이션(ensure_column) |
| UI 위치 | management 신규 탭 | 재고 탭과 동일 권한·맥락, stock.js 패턴 재사용 |
| 발주 처리 | CSV 내보내기까지만 | 외부 발주 전송은 통합 리스크 큼 → Out of Scope |

## 7. Open Questions (기본값으로 자율 결정)

1. 안전버퍼 계수 → 리드타임의 50% 추가버퍼로 `soon` 판정 (기본값 확정)
2. 분석기간 기본값 → 30일 (현장 월 단위 발주 주기와 정합)
3. 단위 혼재(weight/count) → `unit_type='weight'` 자재만 예측, count형은 표시 제외 (기본값 확정)
