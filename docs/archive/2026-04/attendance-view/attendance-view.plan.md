# Attendance View Plan

> 월별 근태 엑셀 파일을 조회하는 사번 인증 기반 페이지 계획서

## 1. Overview

| Item | Detail |
|------|--------|
| Feature | attendance-view |
| Priority | Medium |
| Base | IRMS v0.2 + 기존 entry 진입 구조 |
| Goal | 현장/서버 PC에서 사번 인증 후 본인 근태 확인, 관리자는 전원 조회 가능 |
| Source | `C:\ErpExcel\monthly_attendance_YYYY-MM.xlsx` (매일 18시 ERP에서 갱신) |

## 2. Problem Statement

ERP에서 내려받는 월별 근태 엑셀을 현장 작업자들이 열어볼 수 없어 관리자에게 일일이 문의해야 한다. 엑셀 파일은 서버 PC 고정 경로에 있지만 공유되지 않고, 작업자 23명이 각자 자기 출퇴근·지각·연장 시간을 확인할 수 없다.

### Pain Points

1. 작업자가 본인 근태를 직접 확인할 수 없음
2. 관리자가 개별 문의에 일일이 응답
3. 근태 이상(지각/무출근/무퇴근)이 있어도 본인이 모르고 지나감

## 3. Feature Items

### 3.1 사번 기반 별도 인증 시스템

| Item | Detail |
|------|--------|
| 목표 | 기존 IRMS 계정(manager/operator)과 분리된 근태 전용 인증 |
| ID | 사번 (엑셀 사번 컬럼, 6자리 숫자) |
| 초기 비밀번호 | 사번과 동일 |
| 자동 계정 생성 | 첫 로그인 시 사번이 엑셀에 존재하면 자동 생성 |
| 첫 로그인 비밀번호 변경 강제 | 초기 비밀번호 감지 시 변경 페이지로 리다이렉트 |
| 세션 | irms_att_session 쿠키, **5분 비활성 타임아웃** |
| 로그인 실패 잠금 | **5회 실패 → 5분 잠금** |
| 비밀번호 저장 | bcrypt 해시 (기존 auth.py 방식 재사용) |

### 3.2 관리자 특권 모드 (자동 진입)

| Item | Detail |
|------|--------|
| 조건 | IRMS에 admin/manager 권한으로 이미 로그인된 상태 |
| 동작 | /attendance 접속 시 사번 로그인 생략, 전체 조회 모드로 진입 |
| UI | 전직원 드롭다운(사번+성명) → 선택 → 해당 사번 근태 표시 |
| 감사 로그 | audit_logs 테이블에 `attendance_viewed_by_admin` 기록 |

### 3.3 근태 조회 페이지

| Item | Detail |
|------|--------|
| 위치 | `/attendance` |
| 상단 프로필 | 성명 · 사번 · 부서명 · 공장 · 근무타임 (엑셀에서 매 조회 시 반영) |
| 월 전환 | 상단 `< 2026-04 >` 화살표 버튼, 파일 없는 달은 비활성 |
| 요약 카드 | 근무일, 지각/조퇴/외출 횟수+시간, 평일/휴일 × 정상/연장/야근/조출 (8개) |
| 일자별 테이블 | 일자·요일·구분, 출근·퇴근·익일, 정상/연장/야근/조출/지각/조퇴/외출, 비고 |
| 총 근무시간 통합 합계 표시 금지 | 연장/야근 1.5배 급여 가산이라 오해 유발 |

### 3.4 메인 진입 화면에 근태 카드 추가

| Item | Detail |
|------|--------|
| 위치 | `entry.html` access-grid, 계량·관리 카드 옆 세 번째 카드 |
| 링크 | `/attendance` |
| 버튼 | "근태 확인" 또는 "내 근태" |

### 3.5 관리자 비밀번호 초기화

| Item | Detail |
|------|--------|
| 위치 | 기존 admin_users 또는 별도 "근태 사용자" 섹션 |
| 기능 | 근태 계정 목록 조회 + 선택한 사번의 비번을 사번으로 초기화 |
| 동작 | 초기화 시 해당 사용자는 다음 로그인에서 비번 변경 강제 |

## 4. Scope

### In Scope
- `/attendance` 페이지 (로그인, 비번변경, 조회, 관리자 모드)
- 엑셀 파서 (월별 파일 자동 탐색, read_only로 락 회피)
- 별도 인증 테이블 `attendance_users` (bcrypt 해시)
- 세션 관리 (비활성 타임아웃, 실패 잠금)
- 관리자 특권 자동 감지
- entry.html 세 번째 카드
- 관리자 비번 초기화 버튼
- 감사 로그 (관리자 조회 기록)

### Out of Scope (v1)
- 근태 이상 알림 팝업 (지각/무출근) — 향후 별도 기능
- 트레이 앱 연동 (공지방 경유로 알림) — 향후
- 엑셀 쓰기 (근태 수정) — 조회 전용
- 그래프/차트 시각화 — 테이블만
- 엑셀 파일 외 데이터 소스 — ERP 엑셀에 의존

## 6. Success Criteria

1. 현장 PC에서 `/attendance` 접속 → 사번 로그인(초기 비번=사번) → 비번 변경 강제 → 본인 근태 조회
2. 5분 비활성 → 자동 로그아웃, 5회 실패 → 5분 잠금
3. 관리자로 IRMS 로그인 후 `/attendance` 접속 → 사번 입력 없이 전직원 드롭다운 조회
4. 월 화살표 클릭 시 이전/다음 달 파일로 전환 (파일 없으면 비활성)
5. 엑셀 파일이 ERP로 업데이트되면 다음 조회부터 자동 반영 (매 요청 파일 읽음)
6. 동명이인 문제 없이 사번으로 정확히 구분

## 7. Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| 엑셀 파일 락 (ERP가 잠금) | `openpyxl read_only=True` + 실패 시 "잠시 후 다시 시도" 메시지 |
| 동명이인 | 사번 기반이라 문제 없음 |
| 사번 6자리 brute-force | 5회 실패 잠금 + 내부망 전용 |
| 초기 비번(=사번) 유지 취약성 | 첫 로그인 시 변경 강제 |
| 엑셀 포맷 변경 | 컬럼 인덱스 기반 파서 + 헤더 검증, 실패 시 명확한 에러 |
| 월 전환 시점 파일 없음 | 친절한 "이번 달 파일이 아직 없습니다" 메시지 |

## 8. Implementation Order

1. DB: `attendance_users` 테이블 + 마이그레이션
2. 엑셀 파서 서비스 (월 파일 탐색, 파싱, 집계)
3. 근태 인증 로직 (비번 해시, 세션, 잠금)
4. API 라우터 (로그인/변경/조회/관리자 조회/초기화)
5. 프론트 페이지 (로그인/비번변경/조회/관리자 드롭다운)
6. entry.html 카드 추가
7. 관리자 비번 초기화 UI
8. 스모크 테스트, 커밋, 푸시

## 9. Future (out of scope but planned)

- 주/월 누적 그래프 시각화
- 개인 목표 대비 연장 시간 누적 경고
