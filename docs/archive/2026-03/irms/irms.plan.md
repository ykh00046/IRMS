## 1. 개요 및 목적

### 1.1 배경

현재 잉크 레시피(원재료 투입 배합표)를 **수기 또는 엑셀**로 관리하고 있어 다음과 같은 문제가 발생:

| 문제           | 영향                                                |
| -------------- | --------------------------------------------------- |
| 오기입         | 잘못된 원재료/중량 기록으로 제조 불량               |
| 추적 불가      | 누가/언제/어떤 배합을 했는지 사후 확인 어려움       |
| 안료 오염      | 여러 색상 원료를 동시에 보며 작업 시 교차 오염 위험 |
| 재고 파악 지연 | 소모량을 수동 취합해야 해서 발주 지연               |

### 1.2 목적

사내 **잉크 레시피 관리 시스템(IRMS)**을 구축하여:

- 엑셀 붙여넣기(Zero-Entry)로 레시피 입력 자동화
- 컬러 포커스(Focus Mode)로 안료 교차 오염 방지
- 완료된 레시피의 원재료 소모량 자동 집계

### 1.3 핵심 컨셉

| 컨셉                   | 설명                                          |
| ---------------------- | --------------------------------------------- |
| **Workflow-Centric**   | 현장 작업 동선에 맞춘 가로형 대시보드         |
| **Contamination-Free** | 안료 교체 시 오염을 방지하는 Color Focus Mode |
| **Zero-Entry**         | 엑셀 붙여넣기를 통한 입력 자동화              |

## 2. 범위 (Scope)

### 2.1 In-Scope (Phase 1 — PoC)

| 구분                    | 항목                                                        |
| ----------------------- | ----------------------------------------------------------- |
| **[화면 1] Work Mode**  | 가로형 대시보드, Color Focus Mode, 완료 처리                |
| **[화면 2] Management** | Smart Import(엑셀 붙여넣기), Validation(3단계), 레시피 조회 |
| **[화면 3] Insight**    | 원재료 소비 통계 (Weight 합산 / Count 횟수), 엑셀 Export    |
| **Master DB**           | 원재료 관리 (등록/수정/비활성화), Alias 매핑                |
| **사용자 관리**         | 2역할(Admin/User), ID/PW 인증, 세션 기반                    |
| **감사 로그**           | 핵심 이벤트 Audit Log 기록                                  |
| **운영 환경**           | 로컬 PC Docker 서버, SQLite, 사내 폐쇄망                    |

### 2.2 Out-of-Scope (Phase 2 이후)

- 바코드/라벨 출력
- 원재료 재고 연동 (차감/안전재고/발주)
- SSO 인증 연동
- 중앙 서버 전환 (PostgreSQL)
- 사용자 정의 색상그룹 확장

## 4. 레시피 상태 관리

### 4.1 상태(State) 정의

| 상태        | 의미                   | Work 노출 | 수정 가능 |
| ----------- | ---------------------- | :-------: | :-------: |
| Draft       | 등록 중 (검증 전/임시) |     ✕     |     ○     |
| Pending     | 현장 대기 (작업 전)    |     ○     |  ○(제한)  |
| In-Progress | 작업 진행 중           |     ○     |  ✕(원칙)  |
| Completed   | 작업 완료 (LOCK)       |     ✕     |     ✕     |
| Canceled    | 취소 (사유 필수)       |     ✕     |     ✕     |

### 4.2 상태 전이 규칙

```
[엑셀 붙여넣기]
       │
       ▼
    Draft ──────→ Pending ──────→ In-Progress ──────→ Completed
   (선택)        (검증통과)       (작업시작)           (완료/LOCK)
                    │                │
                    └───→ Canceled ←─┘
                         (사유 필수)
```

- `Draft → Pending` : 검증 통과 시 (Draft 생략 가능)
- `Pending → In-Progress` : 작업 시작 (작업자 자동 기록)
- `Pending → Canceled` : 등록 취소 (사유 필수)
- `In-Progress → Completed` : 완료 (2단 확인)
- `In-Progress → Canceled` : 작업 중 취소 (사유 필수)
- `Completed → 변경 불가` : 오류 시 Revision(신규 레시피) 생성

## 7. Smart Import 규칙

### 7.1 헤더 표준화

- 공백/특수문자 제거 + 대문자 통일 (예: `byk199` → `BYK-199`)
- Alias 매핑: 관리자가 동의어 설정 (예: `GLY` → `GLYCEROL`)
- 필수 컬럼: `제품명`, `위치(1도/2도/3도)`, `잉크명`

### 7.2 Validation 3단계

| 단계    | 정책                       | 결과              |
| ------- | -------------------------- | ----------------- |
| Level 1 | 미등록 원재료              | ERROR (등록 차단) |
| Level 2 | unit_type 불일치           | ERROR             |
| Level 3 | 필수 원료 누락 / 범위 이탈 | WARN              |

### 7.3 엣지 케이스

| 상황                | 처리                       |
| ------------------- | -------------------------- |
| 빈 셀               | Null 처리                  |
| 병합 셀             | 해제 후 파싱, 불가 시 WARN |
| "1.5kg" (숫자+문자) | 숫자 추출 + WARN           |
| 음수                | ERROR                      |
| 이상치 (>10000)     | WARN 확인 요청             |
